# User Personas & User Stories

## Persona 1 — 大学/研究生学生（Alex）
- 年龄：20-26
- 职业：本科/研究生，课程作业与学习为主
- 目标：快速生成高质量写作或编程提示，节省时间，提高作业表现
- 痛点：不会写出能得到有用结果的 Prompt；缺少评估优化后的对比指标

User Stories:
1. 作为学生，我想把作业题目输入系统，获得一个明确、可执行的 Prompt 模板，这样我能快速得到可用答案。
   - 验收标准：系统返回的 Prompt 包含输入、输出示例和约束条件，用户可直接复制使用。
2. 作为学生，我想保存 Prompt 版本并查看每次优化的质量评分，这样我可以跟踪改进效果。
   - 验收标准：Prompt 历史记录可查看，且每个版本有评分与简短说明。
3. 作为学生，我想看到按学科或场景分类的模板库，能快速找到适合我任务的模板。
   - 验收标准：模板库按标签筛选，展示示例与适用场景。

## Persona 2 — AI 初学者 / 非技术用户（Lina）
- 年龄：25-45
- 职业：设计师、市场或行政人员
- 目标：利用 AI 完成内容创作（文案、摘要、市场邮件）而不懂代码
- 痛点：对 Prompt 语法陌生，不知道怎样把想法转成高质量 Prompt

User Stories:
1. 作为非技术用户，我想通过简单的向导输入需求，系统生成可运行的 Prompt，这样我能不懂技术也使用 AI。
   - 验收标准：提供逐步向导（问题+选项），最终生成一段可复制的 Prompt。
2. 作为非技术用户，我希望系统解释每个 Prompt 的作用与改进建议，这样我能学习如何写更好的 Prompt。
   - 验收标准：每个优化结果附带一段易懂的解释（中文），包含改进要点。
3. 作为非技术用户，我想看到不同风格/语气的示例（正式/友好/营销），以便选择合适的输出风格。
   - 验收标准：示例面板支持风格切换并展示对应输出样例。

## Persona 3 — 开发者 / 工程师（Marco）
- 年龄：22-40
- 职业：软件工程师、数据科学家
- 目标：将 Prompt 优化流程集成到产品或自动化流程中
- 痛点：需要可重复、可评估的 Prompt 管道和 API 支持

User Stories:
1. 作为工程师，我希望通过 API 批量提交 Prompt 并获取质量评分，这样可以把 Prompt 优化纳入 CI 流程。
   - 验收标准：提供 REST API（或 SDK）端点，支持批量提交并返回 JSON 评分。
2. 作为工程师，我需要导出 Prompt 历史和评分数据（CSV/JSON），用于离线分析和报表。
   - 验收标准：可在界面或 API 中导出包含 prompt_id、版本、评分、时间戳 的文件。
3. 作为工程师，我想对比两个 Prompt 的 A/B 输出效果并保存实验结果用于回溯。
   - 验收标准：提供 A/B 测试视图，记录输入、模型、输出和评价指标。

## Persona 4 — 内容创作者 / 营销（Sara）
- 年龄：24-45
- 职业：自由撰稿人、内容营销
- 目标：高效产出多版本文案并快速迭代
- 痛点：需要可复用的模板和快速生成不同变体

User Stories:
1. 作为内容创作者，我想一键生成 5 个文案变体并按吸引力排序，这样我能挑选最佳内容。
   - 验收标准：生成多变体并基于可解释的评分指标排序展示。
2. 作为内容创作者，我想存储并标注高质量模板，团队成员可共享使用。
   - 验收标准：模板收藏与共享功能可用，并支持权限控制（团队/公开）。

---

## 共通验收标准建议
- 每条用户故事需有明确的可验证输出（文件/API/界面组件）。
- 所有导出与共享功能需记录时间戳与作者信息。
- 用户行为数据（如模板使用次数、优化成功率）记录并可导出。
